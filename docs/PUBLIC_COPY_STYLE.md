# Public copy style

Positioning: Check that an agent cannot use the wrong account before it touches
a customer, client, or production repository.

Audience: A small engineering or operations team running agents across more
than one brand, client, or company account.

Buying moment: The team can make an agent useful, but cannot answer exactly
which account it will act as when several credentials are available.

Primary action: Run the free check, then book a fixed-scope assessment if a
real deployment needs review.

Voice traits:

- Plain and technical.
- Specific about the boundary being checked.
- Calm about risk. No disaster language.
- Short before the first action; details live below it.

Prefer: check, map, test, keep, isolate, fix, rerun, account, action, report.

Avoid: AI safety theater, platform claims, compliance claims, superlatives,
generic "security" language, and claims of adoption or certification.

Required exact terms: identity, principal, provider, lease, allowlist,
Identity Boundary Assessment.

Homepage and offer order:

1. Name the wrong-account failure.
2. Name the check and its output.
3. State the fixed scope, price, and delivery.
4. Put technical architecture and limitations after the purchase decision.

Examples:

- Before: "Comprehensive security for modern AI agents."
  After: "Check which account your agent can use."
- Before: "A powerful audit of your AI stack."
  After: "Map the accounts, test the boundary, and get a rerun command."
